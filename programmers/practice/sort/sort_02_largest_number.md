# Sort 02 - Largest Number

## Problem

- Platform: Programmers
- Category: Sort
- Title: Largest Number
- Korean title: 가장 큰 수

## Problem Summary

0 또는 양의 정수 배열 `numbers`가 주어진다.

숫자들의 순서를 재배치해서 이어 붙였을 때 만들 수 있는 가장 큰 수를 문자열로 반환해야 한다.

예를 들어:

```text
[6, 10, 2]
```

로 만들 수 있는 가장 큰 수는:

```text
"6210"
```

이다.

## Quick Summary

- 이 문제는 전형적인 sort comparator 문제이다.
- 숫자 자체의 크기순으로 정렬하면 안 된다.
- 문자열로 바꾼 뒤, 두 수 `a`, `b`에 대해 `a + b`와 `b + a`를 비교해야 한다.
- `a + b`가 더 크면 `a`가 앞에 와야 한다.
- 모든 수가 0인 경우 `"000"`이 아니라 `"0"`을 반환해야 한다.

## Constraints

- `numbers`의 길이는 1 이상 100,000 이하이다.
- `numbers`의 원소는 0 이상 1,000 이하이다.
- 정답이 너무 클 수 있으므로 문자열로 반환한다.

## My Approach

처음에는 앞자리 숫자가 큰 순서대로 정렬하면 된다고 생각했다.

예시:

```text
[3, 30, 34, 5, 9]
```

앞자리 기준으로 보면:

```text
9 -> 5 -> 3
```

처럼 볼 수 있다.

그리고 앞자리가 같은 `3`, `30`, `34`는 뒤자리를 비교해서 다음처럼 정렬하면 된다고 생각했다.

```text
34 -> 3 -> 30
```

그래서 최종 순서는:

```text
9 5 34 3 30
```

이 된다.

이 예시에서는 맞는 결과가 나온다.

```text
9534330
```

## Thinking Process

내 초기 생각은 “앞에 왔을 때 더 큰 수를 먼저 둔다”에 가까웠다.

하지만 이를 직접 구현하려고 하면 앞자리, 최대 길이, 뒤자리 비교 같은 규칙이 복잡해진다.

이 문제는 그런 규칙을 직접 만들기보다, 두 수의 상대적 순서를 비교하면 된다.

두 문자열 `a`, `b`가 있을 때:

```text
a + b
b + a
```

중 더 큰 쪽이 되는 순서를 선택한다.

예를 들어 `3`과 `34`를 비교하면:

```text
"3" + "34" = "334"
"34" + "3" = "343"
```

`343`이 더 크므로 `34`가 `3`보다 앞에 와야 한다.

`3`과 `30`을 비교하면:

```text
"3" + "30" = "330"
"30" + "3" = "303"
```

`330`이 더 크므로 `3`이 `30`보다 앞에 와야 한다.

따라서 `[3, 30, 34, 5, 9]`는 다음 순서가 된다.

```text
9 5 34 3 30
```

## Mistakes And Lessons

초기 접근은 앞자리 숫자를 기준으로 생각했다.

예시에서는 맞았지만, 일반화하려면 복잡하다.

실패 지점은 다음이다.

- 숫자 크기 비교 문제가 아니다.
- 앞자리만 비교하는 문제도 아니다.
- 최대 길이에 맞춰 늘려 비교하는 방식은 구현이 복잡하고 실수 여지가 있다.
- 두 수의 순서는 `a + b`와 `b + a`를 비교하면 바로 결정된다.
- 전부 0인 케이스를 놓치면 `"000"`을 반환하게 된다.

따라서 이 문제의 핵심은 직접 규칙을 세분화하는 것이 아니라, sort에 정확한 비교식을 주는 것이다.

정렬 후 첫 값이 `"0"`이면 전체가 0이라는 뜻이다.

```cpp
if (nums[0] == "0") {
    return "0";
}
```

## Key Point

정렬 기준은 다음이다.

```cpp
return a + b > b + a;
```

의미:

```text
a를 b보다 앞에 붙였을 때 더 큰 수가 되면 a를 앞에 둔다.
```

이 문제는 숫자 정렬이 아니라 문자열 조합 결과를 기준으로 정렬하는 문제이다.

## Accepted Solution

```cpp
#include <string>
#include <vector>
#include <algorithm>

using namespace std;

string solution(vector<int> numbers) {
    vector<string> nums;
    for (auto number : numbers) {
        nums.push_back(to_string(number));
    }

    sort(nums.begin(), nums.end(), [](const string& a, const string& b) {
        return a + b > b + a;
    });

    if (nums[0] == "0") {
        return "0";
    }

    string answer = "";
    for (auto num : nums) {
        answer += num;
    }

    return answer;
}
```

## Complexity

숫자의 개수를 `n`이라고 하면 정렬이 핵심이다.

정렬은 `O(n log n)`번 비교를 수행한다.

각 비교에서 문자열 결합 `a + b`, `b + a`가 일어나지만, 문제에서 각 숫자는 1,000 이하라 문자열 길이는 작다.

따라서 전체 시간복잡도는 `O(n log n)`으로 볼 수 있다.

공간복잡도는 문자열 배열을 저장하므로 `O(n)`이다.

## Notes

- `sort` 비교 함수에서 `true`를 반환하면 첫 번째 인자 `a`가 앞에 온다.
- 이 문제는 `sort`에 커스텀 비교식을 주는 것이 핵심이다.
- 결과가 전부 0인 경우를 처리해야 한다.
- 정렬 후 첫 문자열이 `"0"`이면 모든 값이 0이므로 `"0"`을 반환한다.
