# Algorithm

알고리즘 문제 풀이를 기록하는 저장소입니다.

문제를 풀면서 떠올린 접근, 실패한 이유, 최종 풀이, 복습 포인트를 함께 남깁니다. 정답 코드만 저장하기보다 다음에 같은 유형을 봤을 때 어떤 신호를 보고 어떤 패턴을 떠올릴지 정리하는 것을 목표로 합니다.

## Note Structure

문제 기록은 가능한 한 아래 순서로 작성합니다.

```text
Problem
Problem Summary
Quick Summary
Constraints
My Approach
Thinking Process
Mistakes And Lessons
Key Point
Accepted Solution
Alternative Solution
Complexity
Notes
```

없는 항목은 생략할 수 있지만, 복습하기 쉽도록 섹션 이름과 순서는 최대한 유지합니다.

## Study Rules

- 처음 생각한 풀이를 기록한다.
- 틀린 이유와 놓친 조건을 기록한다.
- 최종 코드는 내가 이해한 방식으로 남긴다.
- 다른 풀이가 있으면 `Alternative Solution`으로 분리한다.
- 시간복잡도와 공간복잡도를 확인한다.
